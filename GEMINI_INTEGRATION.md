# Google Gemini API Integration

## Overview

The backend AI service has been successfully integrated with **Google Gemini API**, removing the dependency on local Ollama/Qwen instances. All existing functionality, database workflows, and memory systems remain untouched and fully operational.

## Changed Files

### 1. **artifacts/api-server/package.json**
- **Change**: Added `@google/generative-ai` (v0.21.0) dependency
- **Reason**: Official Google SDK for accessing Gemini models programmatically
- **Impact**: Single new dependency - no breaking changes to existing packages

### 2. **artifacts/api-server/src/lib/aiService.ts** (Primary Integration Point)
- **Complete Refactor**: Replaced Ollama HTTP-based queries with Gemini API calls
- **Key Changes**:
  - Removed: `AI_BASE_URL` (Ollama endpoint)
  - Added: `GEMINI_API_KEY` initialization and validation
  - Added: `MAX_RETRIES` (2) with exponential backoff retry logic
  - Added: `INITIAL_RETRY_DELAY_MS` (500ms) for retry timing

#### Functions Updated:

##### **checkAiOnline()**
- Old: Checked `/api/tags` and `/health` endpoints on local Ollama
- New: Validates Gemini API connectivity via `countTokens()` call
- Returns same interface: `{online, model, provider: "gemini", responseTimeMs, error}`
- Logs: Health check failures to backend logs

##### **queryAi(prompt: string)**
- Old: POST request to `AI_BASE_URL/api/generate` with streaming disabled
- New: Uses Gemini `generateContent()` with:
  - **Temperature**: 0.25 (low - stable, analytical responses)
  - **maxOutputTokens**: 500 (market analysis conciseness)
  - **topP**: 0.9 (nucleus sampling - high consistency)
  - **safetySettings**: All set to BLOCK_NONE (market analysis requires unrestricted content)
  - **Timeout**: 25 seconds (AbortController + Promise.race)
  - **Retry Logic**:
    - Up to 2 retries on failure
    - Exponential backoff: 500ms → 1000ms
    - Full logging of each attempt

#### New Logging:
- **On Success**: `{ model, latencyMs, attempt, provider: "gemini" }`
- **On Retry**: Warnings with error details before retry
- **On Final Failure**: Full error context logged before fallback to rule-based analysis

### 3. **artifacts/api-server/.env.example**
- **New File**: Created configuration template
- **Contains**:
  ```
  DATABASE_URL=postgresql://...
  GEMINI_API_KEY=your_gemini_api_key_here
  AI_MODEL=gemini-2.5-flash
  PORT=3000
  NODE_ENV=development
  ```

## Setup & Configuration

### Step 1: Get Gemini API Key

1. Visit [Google AI Studio](https://aistudio.google.com/app/apikeys)
2. Click "Create API Key"
3. Copy your key (looks like: `AIza...`)

### Step 2: Configure Environment

Create or update `.env` in `artifacts/api-server/`:

```bash
GEMINI_API_KEY=AIza...your_actual_key...
AI_MODEL=gemini-2.5-flash
DATABASE_URL=postgresql://user:password@localhost:5432/market_intelligence
PORT=3000
NODE_ENV=development
```

### Step 3: Install Dependencies

From workspace root:

```bash
pnpm install
```

This installs `@google/generative-ai` in the api-server package.

### Step 4: Run Backend

From workspace root:

```bash
pnpm --filter @workspace/api-server dev
```

Or directly in `artifacts/api-server/`:

```bash
pnpm dev
```

Expected output:
```
API Server running on http://localhost:3000
Gemini API initialized successfully
```

## Fallback Behavior

If Gemini API is unavailable or GEMINI_API_KEY is missing:

1. Health check returns `{ online: false, error: "..." }`
2. Analysis requests automatically fall back to **rule-based analysis**
3. Rule-based analysis uses:
   - Technical indicators (RSI, MACD, EMA)
   - Signal merging
   - Pattern detection
   - Historical memory system
4. **No data loss** - learning/memory systems continue operating
5. Analysis quality degrades gracefully (confidence scores adjusted)

## Troubleshooting

### ❌ Error: "GEMINI_API_KEY not configured"
**Solution**: Ensure `.env` file in `artifacts/api-server/` has `GEMINI_API_KEY=...`

### ❌ Error: "Request timeout"
**Solution**: Gemini API took >25 seconds. Check network and API status. System falls back to rule-based analysis.

### ❌ Error: "Invalid API key"
**Solution**: Verify API key is correct and has Gemini API enabled in Google Cloud Console.

### ❌ Requests succeed but analysis quality is lower
**Solution**: Model might be overloaded. Try `gemini-1.5-pro` for better reasoning capacity.

### ✅ Fallback to rule-based analysis happening repeatedly
**Solution**: Check server logs for Gemini API errors. Verify internet connectivity and API quota.
